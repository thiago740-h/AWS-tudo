# Estudo AWS: Conceitos Fundamentais

Este repositório contém uma visão geral técnica sobre a Amazon Web Services (AWS), focando em sua definição, infraestrutura e modelo econômico.

---

## 1. O que é a AWS?
A **Amazon Web Services (AWS)** é a plataforma de computação em nuvem mais adotada e abrangente do mundo. Em vez de gerir hardware físico, a AWS oferece acesso on-demand (sob demanda) a recursos de computação, armazenamento, bases de dados e outras tecnologias através da internet.

**Principais características:**
* **Agilidade:** Acesso instantâneo a centenas de serviços.
* **Inovação:** Ferramentas avançadas de IA, Machine Learning e IoT.
* **Segurança:** Infraestrutura construída para satisfazer os requisitos de segurança mais rigorosos do mundo.

---

## 2. Infraestrutura Global
A AWS está organizada em torno de **Regiões** e **Zonas de Disponibilidade (AZs)** para garantir alta performance e tolerância a falhas.

### Regiões (Regions)
Uma Região é uma localização geográfica isolada no mundo onde a AWS hospeda centros de dados.
* Cada região é composta por múltiplas AZs.
* Permite o cumprimento de leis de soberania de dados locais.
* Reduz a latência ao aproximar os serviços dos utilizadores finais.

### Zonas de Disponibilidade (Availability Zones - AZs)
As AZs são um ou mais centros de dados discretos com energia, rede e conectividade redundantes dentro de uma Região.
* **Isolamento:** São separadas fisicamente por quilómetros para evitar que um desastre natural afete mais do que uma AZ.
* **Alta Disponibilidade:** Aplicações distribuídas por várias AZs permanecem online mesmo que um centro de dados falhe.

## 3. Economia da Nuvem
A AWS revoluciona a forma como as empresas investem em tecnologia através de um modelo flexível.

### Modelo de Precificação: Pay-as-you-go
O modelo **"Pague apenas pelo que usar"** permite que as empresas:
* Substituam despesas de capital (CapEx - hardware caro) por despesas operacionais (OpEx - custos variáveis).
* Eliminem a necessidade de prever a capacidade, evitando o desperdício de recursos ociosos.
* Paguem apenas pelos gigabytes armazenados ou pelas horas de computação utilizadas.

### Economia de Escala
A AWS consegue operar com custos muito baixos devido ao seu volume massivo de clientes.
* **Poder de compra:** A escala permite à AWS adquirir hardware com descontos significativos.
* **Geração de Lucro:** Ao otimizar a utilização dos recursos globais, a AWS reduz o custo marginal.
* **Vantagem para o cliente:** Parte dessas economias é repassada aos clientes na forma de reduções de preços frequentes.
